# 📋 Metodologia de Trabalho

## Vibe Coding + AI-First Development

Minha abordagem combina **Vibe Coding** com **desenvolvimento orientado a IA** para criar produtos digitais em velocidade máxima sem sacrificar qualidade.

---

## 🔄 Ciclo de Desenvolvimento

```
Ideia → MVP funcional (horas) → Feedback → Refinamento → Produto Final
```

### Fase 1: Descoberta & Definição
- Entender o problema de negócio e a persona do usuário
- Definir critérios de sucesso e MVP scope
- Escolher a stack ideal (Lovable vs código vs low-code)

### Fase 2: Prototipação Rápida (Vibe Coding)
- Gerar protótipo funcional com **Lovable** + Supabase
- Validar fluxo e experiência do usuário
- Iterar rapidamente com feedback visual

### Fase 3: Automação & Agentes
- Conectar N8N para workflows de processo
- Implementar agentes de IA para tarefas específicas
- Integrar APIs e serviços externos

### Fase 4: Refinamento & Qualidade
- Aplicar princípios de design (Apple-like adaptado)
- Testar com usuários reais
- Otimizar performance e experiência

### Fase 5: Deployment & Monitoramento
- Publicar com infraestrutura cloud
- Monitorar uso e performance
- Iterar baseado em dados reais

---

## 🛠️ Stack Selection Framework

| Problema | Stack Recomendada |
|---|---|
| Site institucional / Landing page | Lovable + Supabase |
| SaaS completo | Lovable + Supabase + N8N |
| Automação de processos | N8N + Webhooks + IA |
| Agente autônomo | Hermes Agent + LLM API |
| API integration | REST APIs + Webhooks |
| AI Gateway / Proxy | OmniRoute |
| Estudos de design | HTML/CSS + GSAP |

---

## 📐 Princípios de Design (Apple-like Adaptados)

1. **Profundidade visual** — Camadas e texturas em vez de cards planos
2. **Motion significativo** — Animação como parte da narrativa, não enfeite
3. **Hierarquia clara** — Tipografia e espaçamento como elementos de identidade
4. **Simplicidade funcional** — Cada elemento tem um propósito
5. **Consistência** — Sistema de design unificado em todas as plataformas

## ✅ Garantia de Qualidade

- **Verificação de cada resultado** — Nunca assumir que funcionou sem testar
- **Documentação** — Cada projeto com README claro
- **Versionamento** — Código e configurações no GitHub
- **Métricas** — Onde possível, resultados quantificados
